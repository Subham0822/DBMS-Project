# **App Name**: MediSys

## Core Features:

- Role-Based Dashboards: Separate dashboards for Admin, Doctor, and Patient users with unique layouts and color highlights, role-based navigation with icons, sidebar menus, and smooth transitions between pages.
- Patient Management: Simulated CRUD operations for patient records. Display cards or table views showing patient info. Animated modals for record creation and editing.
- Doctor Management: Simulated CRUD for doctors’ profiles, specialties, and schedules. Include small profile cards with photo/avatar, specialization, and availability. Show appointment slots visually on a mini schedule grid.
- Appointment Booking: Patients can search available doctors (by specialty or name). Appointment booking form with date/time pickers. Show a simulated confirmation message and add the appointment to the dashboard list.
- Billing Simulation: Generate and display mock billing records (amount, date, payment status). Use status badges. Include download/view mock PDF options for bills.
- Medical Record Access: Doctors can view patient medical histories and add new notes/diagnoses. Store all in-memory for MVP. Display medical records in clean, card-based or tabbed views.
- AI-Driven Symptom Checker: Add an interactive AI symptom checker tool (text-based). Patients can enter symptoms and the tool gives likely causes (simulated response). Visually engaging interface (chat-style or modal with animation).

## Style Guidelines:

- Primary: Calm Blue #5DADE2 (trust & professionalism)
- Background: Light Gray #ECF0F1 (clean & spacious)
- Accent: Soft Green #A2D9CE (for positive actions & highlights)
- Error/Alert: Soft Red #E74C3C
- Font: 'PT Sans' for body and headings. Large, clear section headings. Good line spacing and hierarchy.
- Use flat, minimalist icons for medical services. Prefer rounded, friendly iconography — no harsh edges.
- Responsive and mobile-friendly. Sidebar navigation with hover animation. Card-based data presentation. Charts/Stats widgets for summaries. Dashboard-style layout similar to a startup SaaS app.
- Smooth transitions & micro-animations. Light motion transitions when switching between dashboards. Animated modals for CRUD actions. Loading spinners or skeleton screens for a modern touch. Subtle hover effects on cards and buttons.
